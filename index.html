  • I am not an expert. I'm very much a beginner

  • I'm just a person who started learning Idris, got excited and wanted to get other people excited.

"Idris is a general purpose pure functional programming language with dependent types."

http://idris-lang.org
  • Like Haskell, Idris is pure, meaning functions can't have side-effects.
  • Unlike Haskell, Idris has dependent types. Types can depend on values. For example, you can write a function that combines two lists. In most langs we can specify that each list must contain the same types of items. In Idris we can say that, given a list `n` length and a list of `m` length, the function returns a list of `n + m` length

    • What Makes Idris Unique

  • Or more accurately, what makes Idris worth your time.
  • It mostly looks like Haskell, but with a few extras. So what

    • First-Class Dependent Types

    • First-class means there are no syntactic restrictions.
    • Functions can comput types, types can contain values (as we saw).
    • Anything you can write at the value level, you can write at the type level.

    Expressiveness

    • Idris encourages a style of development that puts as much emphasis on coming up with adequately precise types as it does on writing functions.
    • Describe a problem with as much precision as possible, and encode that into the types.

    Sudoku

    • A 9*9 grid of numbers
    • A 9*9 grid of positive integers
    • A 9*9 grid of positive integers between 1 and 9
    • A 9*9 grid, with 9 3*3 boxes, containing positive integers between 1 and 9, where each integer appears exactly once in each row, column, and box
    • Each of this describes a Sudoku game.
    • But only the last is precise enough to make illegal states unrepresentable.

    All The Precision?

    no.

    • Like most things in software there are tradeoffs
    • Precise types reduce & possibly completely remove certain classes of bugs from your code.
    • Too precise though and they become difficult to write, read and maintain.

    Type-Driven Development

    Type -> Define -> Refine
    • Identify the data to be represented
    • Design a type to represent that data, and decide what aspects of that data should be encoded in the type.
    • Identify an operation to act on that data
    • Establish the function input and output and decide if there there is enough precision. If not return to step 2.
    • Implement the function. If difficult reconsider your type.
    • Use the compiler as a tool to guide you

    Compiler As An Assistant

    • In Idris the compiler isn't just there to tell you when you make an error.
    • The compiler can actively assist you in writing your program. The trick is getting types that expressive enough.
